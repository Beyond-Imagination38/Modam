# Project-Design & Implementation
| 항목 | 내용 |
|:---  |---  |
| **요구사항 정의**<br> **UI 분석/설계 모델**<br> **1. 회원가입 / 로그인 화면**<br> - **UI 요소** 이메일, 비밀번호 입력 필드, 로그인, 가입하기 버튼<br> - **기능 연관** 사용자 인증 → `user` 테이블에 사용자 정보 저장, 로그인 후 시스템에 접근할 수 있도록 인증 처리<br> **2. 모임 상세 페이지**<br> - **UI 요소** 모임 소개 텍스트, 모임 일정, 참여자 정보 표시, 참여 신청 버튼<br> - **기능 연관** `bookclub` 테이블에서 모임 정보 조회 및 관리, `participants` 테이블을 통해 사용자 참여 상태 업데이트<br> **3. 채팅 화면 (실시간 채팅 독서모임)**<br> - **UI 요소** 채팅 메시지 입력창, 실시간 메시지 출력 영역, 메시지 필터링 기능 (비속어, 혐오 표현 처리)<br> - **기능 연관** `chatmessage` 테이블에 메시지 저장, `meetingprogress`와 연동되어 발언 순서 관리, 비속어 및 혐오 표현 필터링 기능 구현<br> **4. 발제문 선택**<br> - **UI 요소** AI가 생성한 발제문 선택 옵션<br> - **기능 연관** `discussiontopic` 테이블에 저장된 발제문을 사용, `version` 필드를 통해 각 모임에 대한 발제문 순서 관리<br> **5. 메모 작성 / 독후감 제출**<br> - **UI 요소** 텍스트 입력창 (메모, 독후감), 임시 저장 및 제출 버튼<br> - **기능 연관** `memo`, `readingnote` 테이블에 데이터 저장<br> **6. 토론 요약**<br> - **UI 요소** 토론 요약문 표시 영역<br> - **기능 연관** `bookclub` 테이블에 AI가 생성한 요약 저장<br> 📌 **UI 흐름 및 주요 기능 연관** `[로그인] → [홈 화면] → [모임 상세] → [참여 신청] → [채팅 화면] ↓ ↑ [회원가입] → [모임 목록] → [발제문 선택] → [메모 작성/독후감 제출] ↓ [모임 진행 및 피드백] → [토론 요약 확인] → [독후감 첨삭 요청]`<br> **UI 설계 및 기능 연계 분석**<br> - **회원가입 및 로그인 화면**에서 사용자는 인증 정보를 입력하고, 이후 **홈 화면**과 **모임 상세 페이지**로 이동.<br> - **모임 상세 페이지**에서 사용자는 `bookclub` 테이블에서 제공하는 모임 정보를 확인하고, 참여를 신청.<br> - **채팅 화면**에서 실시간으로 메시지를 주고받으며, `chatmessage` 테이블에 메시지가 저장됨.<br> - **발제문 작성 및 선택 화면**에서는 AI 생성 발제문이 `discussiontopic` 테이블에 저장되고, 사용자는 이를 선택하여 토론을 시작.<br> - **메모 작성 및 독후감 제출** 기능은 각각 `memo`와 `readingnote` 테이블에 연동되어 데이터를 저장.<br> - **토론 요약** 기능은 **AI가 생성한 요약**을 `bookclub` 테이블에 저장하여 나중에 참조할 수 있음.<br> **DATABASE ERD**<br> |
